## 核心功能实现

### 1. 文档生成 ✅

- 使用 OpenAI GPT-4o-mini 模型
- 结构化提示词，确保输出格式一致
- 自动截断过长内容（避免超过 token 限制）

### 2. 并发处理 ✅

- 使用 asyncio 异步处理
- Semaphore 控制并发数（默认 5）
- 避免 API 限流

### 3. 进度显示 ✅

- tqdm 实时进度条
- 显示处理速度和预计剩余时间

### 4. 错误处理 ✅

- 自动重试机制（最多 3 次，指数退避）
- 详细错误日志
- 失败文件跳过，继续处理

### 5. 文件管理 ✅

- 自动检查文件是否已存在
- 跳过已处理文件
- UTF-8 编码处理

### 6. 统计报告 ✅

- JSON 格式报告
- 包含成功/跳过/失败统计
- 详细的处理结果列表

## 使用方法

### 快速开始

```bash
# 1. 配置 API Key
编辑 .env 文件，添加：OPENAI_API_KEY=sk-your-key

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行测试
python test_system.py

# 4. 处理所有文件
python main_simple.py
```

### 自定义配置

在 `main_simple.py` 中修改：

```python
# 并发数（建议 3-5）
max_concurrent=5

# 处理文件数量限制（None = 全部）
limit=None

# 模型选择
model="gpt-4o-mini"  # 或 "gpt-4o"
```

## 故障排除

### 常见问题

1. **API Key 错误**
   - 检查 `.env` 文件中的 API Key 是否正确
   - 确保 API Key 有足够的配额

2. **API 限流**
   - 降低 `max_concurrent` 参数（改为 2 或 3）
   - 增加重试延迟时间

3. **网络错误**
   - 检查网络连接
   - 配置代理（如需要）

4. **编码错误**
   - 确保所有文件使用 UTF-8 编码
   - 检查源文件是否包含特殊字符
